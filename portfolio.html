Case Study 1: OpenGov

The city of Palo Alto's OpenGov website is a way for citizens to have access to their city's financial data. Task: Design a CSV upload process for city official with a way to map their fields to OpenGov fields. Also design a way for an Admin to check upload requests.

Assumptions:

  • User has signed in.
  • There's already an internal landing page where upload link exists.
  • In this workflow, person mapping the data and the Admin are two separate users.
  • Only CSV files are uploaded.
  • The file uploaded in the design has no formatting errors.
  • The file uploaded in the design has missing column headers.

Results:

  • Upload workflow. These wireframes go through the CSV upload and field mapping process. These wireframes are interactive and best viewed when downloaded.
  • Admin Review workflow. After a CSV has been uploaded and mapped, the Admin needs to go in to verify the upload request before it gets pushed into the city dataset.

Case Study 2: MyFedLoan

MyFedLoan.org is a portal to pay back federal student loans. Task: improve their home page (screenshot).

Primary use cases:

  • Signing in
  • Creating an account
  • Contacting MyFedLoan
  • Accessing Help Center

Problems found (annotated screenshot):

  • Primary use cases all over the page (red stars)
  • Redundant information (red)
  • Miscategorized information (purple)
  • Advertising to different types of borrowers in different areas (blue)

Results:

  • Redesigned homepage. The new design highlights the primary use cases for the federal loan payment site. Redundant information has been trimmed and similar information has been grouped.
